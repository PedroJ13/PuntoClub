# TASK-089 - Preparar backlog post-piloto inmediato

## Equipo

Product / Architect / Release

## Contexto

El MVP esta listo para piloto controlado. Despues del primer uso real se necesitara convertir hallazgos en tareas concretas, sin mezclar bloqueos con mejoras futuras.

## Objetivo

Preparar una estructura de backlog post-piloto inmediato para recibir hallazgos y decidir siguiente release.

## Alcance

- Crear categorias:
  - bloqueos P0/P1;
  - mejoras P2 antes de ampliar piloto;
  - pulido P3;
  - post-MVP;
  - decisiones de producto;
  - decisiones operativas Azure/SQL.
- Definir plantilla de tarea derivada de hallazgo.
- Definir criterios para:
  - hotfix;
  - siguiente batch;
  - diferir.
- No crear tareas especificas sin hallazgos reales, salvo placeholders claros.

## No tocar

- No implementar cambios.
- No cambiar Azure.
- No cerrar decisiones sin evidencia de piloto.

## Handoff esperado

Al terminar, actualiza:

```text
tasks/TASK-089-HANDOFF.md
```

Incluye:

- Estructura propuesta.
- Plantilla de tarea.
- Criterios de priorizacion.
- Recomendacion de proceso para Product / Architect / Release.
