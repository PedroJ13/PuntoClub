# TASK-301 - Auditoria UX y direccion visual de polish

Equipo: Ejecucion Tecnica
Modo de ejecucion: Diseno/UX
Round: 42
Depende de: Ninguna
Estado: Assigned
Prioridad: P1 pre-lanzamiento

## Objetivo

Auditar la experiencia actual de Punto Club y definir una direccion UX/visual clara para una pasada fuerte de polish antes de mostrar la app a empresas reales.

## Documentos a leer

- `AGENTS.md`
- `docs/MVP_RELEASE_STATUS.md`
- `docs/NEXT_PHASE_UX_COPY_POLISH.md`
- `chat-start/PRODUCT_ARCHITECT_RELEASE.md`

## Alcance

Revisar las pantallas y flujos publicados/actuales desde criterio UX, sin implementar codigo:

- Registro de empresa.
- Aprobacion/admin de empresas.
- Invitacion y crear acceso.
- Login/logout.
- Atender cliente.
- Puntos: registrar compra, redimir puntos, historial.
- Membresias: activar/renovar/aplicar beneficio.
- Mi empresa: datos, logo, configuracion de puntos y membresias.
- Reportes.

Definir recomendaciones concretas para:

- jerarquia visual;
- densidad de paneles/tarjetas;
- estados vacios;
- mensajes de exito/error;
- acciones principales/secundarias;
- iconos en botones;
- paleta de colores para app de fidelizacion;
- consistencia visual desktop/mobile.

## Criterios de aceptacion

- Entrega una lista priorizada P1/P2/P3 de fricciones UX.
- Define una guia corta de estilo visual para Web Dev.
- Propone que botones deben usar icono + texto y cuales pueden quedar solo texto.
- Propone colores/estados sin redisenar todo desde cero.
- No implementa codigo.

## Handoff esperado

Crear o actualizar `tasks/TASK-301-HANDOFF.md` con:

- Resultado.
- Pantallas revisadas.
- Hallazgos P1/P2/P3.
- Guia visual resumida.
- Recomendaciones para Web Dev.
